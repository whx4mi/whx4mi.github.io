ReconX v5
Pentest Orchestrator · Flask + SocketIO · Pipeline Builder
EM DESENVOLVIMENTO LABORATÓRIO
Plataforma web single-file que orquestra um toolchain ofensivo completo via interface em tempo real. Em vez de abrir terminais separados para cada ferramenta, o ReconX unifica recon, enumeração, análise de vulnerabilidades e brute-force em pipelines configuráveis — com output estruturado, proxy integrado e persistência de findings.

Construído para uso em CTFs e labs de pentest, com foco em velocidade de cobertura e organização de resultados.

// Funcionalidades
Proxy Profiles
Integração nativa com Burp Suite, OWASP ZAP, Tor+Privoxy e proxy custom. Injeção automática nas ferramentas que suportam — sem configurar cada uma manualmente.
Per-tool Timeout
Watchdog com threading.Timer mata processos que excedem o tempo limite. Evita que uma ferramenta lenta trave todo o pipeline. Padrão configurável por ferramenta.
Findings SQLite
Resultados persistidos em banco local com classificação por severidade (Critical, High, Medium, Low). Sessão sobrevive a recarregamentos.
Screenshots automáticos
gowitness captura screenshot de cada endpoint descoberto — aceleração visual da triagem de superfície de ataque.
Multi-port
Detecção de todas as portas HTTP/HTTPS abertas e execução dos módulos web contra cada uma — sem perder serviços em portas não-padrão.
Pipeline Builder
Seleção e ordenação customizada de ferramentas via UI. Pipelines pré-configurados para os cenários mais comuns (web, subdomain, CTF, bug bounty).

// O que aprendi
  • Gerenciamento de subprocessos em Python — controle de processos com timeout, leitura de output em tempo real e tratamento de erros por ferramenta.
  • WebSockets com SocketIO — streaming de output de terminal para o browser em tempo real sem polling.
  • Compatibilidade entre ferramentas — cada ferramenta tem quirks de sintaxe, flags e comportamento. Abstrair isso em um layer unificado exigiu bastante iteração.
  • Proxy como first-class feature — entender quais ferramentas suportam qual tipo de proxy (HTTP vs SOCKS5) e como injetar isso de forma transparente.
  • Design de pipeline — a ordem das ferramentas importa. subfinder antes de httpx, httpx antes de nuclei — a sequência certa multiplica a efetividade.
