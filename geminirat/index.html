GeminiRAT
C2 Framework · Malware Research · Detection Engineering
CONCLUÍDO PRIVADO LABORATÓRIO
Framework de Comando & Controle desenvolvido do zero em Python para emulação de adversários em ambiente de laboratório isolado. O objetivo principal não foi construir um RAT funcional, mas entender como malware real opera — e usar esse conhecimento para derivar regras de detecção eficazes.

O mesmo lab que rodou o GeminiRAT foi monitorado com Zeek + Sysmon + Wazuh, gerando os dados comportamentais que se tornaram Sigma rules e alertas — conectando perspectiva ofensiva e defensiva.
linguagem Python 3 ambiente Kali Linux + Windows VM (VirtualBox) repo privado (revisão sob solicitação)
// Arquitetura
Visão Geral do Sistema
# Componentes principais SERVER (C2 Controller) └── Escuta conexões de agentes └── Interface de comandos interativa └── Gerenciamento de sessões ativas AGENT (Implant — roda na máquina alvo) └── Comunicação via Sockets TCP └── Persistência no Registro do Windows └── Coleta de informações do host └── Exfiltração de dados em staging LAB MONITOR (análise comportamental) └── Zeek → análise de tráfego de rede └── Sysmon → eventos de processo/registro └── Wazuh → correlação e alertas └── Output → Sigma Rules
// Funcionalidades implementadas
Comunicação C2
Canal de comunicação bidirecional via Sockets TCP. Agente mantém conexão persistente e aguarda instruções do servidor controlador.
Persistência
Mecanismo de persistência via Registro do Windows (HKCU\Software\Microsoft\Windows\CurrentVersion\Run) para sobreviver a reinicializações.
Coleta de informações
Enumeração do host: usuário, hostname, SO, processos em execução, interfaces de rede e informações do ambiente.
Exfiltração staged
Transferência de arquivos em chunks (staging) para simular exfiltração real e gerar os artefatos de rede analisados pelo Zeek.
Análise comportamental
Lab monitorado com Zeek (tráfego) e Sysmon (eventos de sistema) durante execução — gerando dataset para derivar regras de detecção.
Detection Engineering
Sigma rules e alertas Wazuh derivados do comportamento real observado — fechando o ciclo red-to-blue com artefatos defensivos concretos.

// O que aprendi
  • Comunicação de rede de baixo nível — implementar Sockets do zero força entendimento real de como implants estabelecem e mantêm canais C2.
  • Superfície de detecção do Windows — persistência via registro deixa rastros específicos e detectáveis. Aprendi quais chaves o Sysmon monitora e como gerar alertas a partir delas.
  • Ciclo red-to-blue — a parte mais valiosa foi usar o próprio lab ofensivo para criar artefatos defensivos. Ver o tráfego do Zeek e os eventos do Sysmon enquanto o agente rodava deu clareza sobre o que os defensores veem.
  • Sigma como linguagem de detecção — traduzir comportamento observado em regras Sigma estruturadas, mapeadas ao MITRE ATT&CK, com lógica de correlação de eventos.
  • Isolamento de lab é inegociável — todo desenvolvimento foi feito em rede isolada (host-only adapter no VirtualBox), sem risco de contaminação de ambiente externo.
